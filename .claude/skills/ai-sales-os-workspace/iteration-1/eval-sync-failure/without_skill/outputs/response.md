# حل مشكلة Sync الفاشل مع رسالة 429 من Apollo

## شرح المشكلة
الخطأ **429** من Apollo يعني أنك تجاوزت حد معدل الطلبات (Rate Limiting). Apollo يحدد عدد الطلبات المسموح بها في فترة زمنية معينة، وعند تجاوزك لهذا الحد، يرفض الطلبات الإضافية مؤقتاً.

---

## الحلول الفورية

### 1. **انتظر قبل إعادة المحاولة**
- الخطأ 429 عادة يكون مؤقت (عادة 15-60 ثانية)
- انتظر 2-5 دقائق على الأقل قبل إعادة تشغيل الـ Sync

### 2. **تحقق من حالة API الخاص بك**
- تأكد من أن مفتاح API (`APOLLO_API_KEY`) صحيح وفعال
- تحقق من أن حسابك على Apollo لم يتم تعليقه أو الحد من وصوله
- ادخل إلى لوحة التحكم Apollo وتحقق من الاستخدام

### 3. **قلل حجم الطلبات**
إذا كنت تشغل sync كامل أو backfill كبير:
- استخدم **النمط الـ Incremental** بدلاً من Full
  ```bash
  python daily_sync.py --mode incremental --hours 24
  ```
- للـ backfill الكبير، قسّمه على عدة أيام:
  ```bash
  python daily_sync.py --mode backfill --days 7
  ```

---

## معالجة الفشل بشكل منهجي

### الخطوة 1: تحليل اللوج
- افتح ملف اللوج (عادة يكون في `logs/` أو في GitHub Actions artifacts)
- ابحث عن السطر الذي يحتوي على `429`
- لاحظ الوقت بالضبط ومقدار الطلبات التي تم إرسالها قبل الفشل

### الخطوة 2: استخدم Checkpoint (إذا كنت تستخدم backfill)
إذا كنت تشغل `--mode backfill`:
- البرنامج يحفظ تقدمه في `backfill_checkpoint.json`
- عند إعادة التشغيل، سيبدأ من آخر نقطة نجح فيها (ليس من البداية)

```bash
python daily_sync.py --mode backfill --days 365
```

### الخطوة 3: فعّل الـ Retry Logic
معظم أدوات Sync الحديثة تتضمن:
- **Exponential Backoff** — محاولة تلقائية مع انتظار متزايد
- تأكد من أن البرنامج الخاص بك يحاول مجدداً (3-5 مرات عادة) قبل الاستسلام

---

## منع المشكلة مستقبلاً

### ✅ أفضل الممارسات

1. **استخدم Incremental Sync**
   - استخدمه يومياً (أسرع وأخف على API)
   - Full sync فقط عند الحاجة الفعلية (مرة شهرياً مثلاً)

2. **حدد وقتاً محدداً للـ Sync**
   - لا تشغّل Full Sync وقت الذروة
   - ستخدم GitHub Actions لجدولة الـ Sync تلقائياً في ساعة محددة

3. **راقب الاستخدام**
   - احفظ نسخة من آخر تقرير استخدام من Apollo
   - إذا كنت قريب جداً من الحد، اتصل بدعم Apollo

4. **أضف تأخير بين الطلبات**
   - بعض البرامج تضيف تأخير صغير (100-500 ملي ثانية) بين الطلبات
   - هذا يقلل احتمال 429


## علامات التحذير الأخرى

| الخطأ | السبب | الحل |
|-------|--------|------|
| **429** | تجاوز معدل الطلبات | انتظر 5 دقائق وأعد المحاولة |
| **401** | مفتاح API غير صحيح | تحقق من `.env` |
| **500** | خادم Apollo معطل مؤقتاً | انتظر 5 دقائق |
| **Timeout** | اتصال بطيء | اختصر حجم البيانات أو أعد المحاولة |
