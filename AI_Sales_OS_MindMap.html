Three Sync Modes

⚡ Incremental

Daily use -- GitHub Actions cron

  • • Date filter (last N hours/days)
  • • Time-window splitting if >50K
  • • Fast -- minutes only
--mode incremental --days 1

📦 Backfill

Historical re-sync with checkpoint

  • • Same logic + progress saved to file
  • • Resumes from where it stopped
  • • For large time ranges (365 days)
--mode backfill --days 365

🔄 Full

Complete rebuild -- no date filter

  • • Alphabetical partitioning (A-Z + numbers)
  • • Pulls every record from Apollo
  • • 2-4 hours -- rare use
--mode full

Decision Log -- Architecture Choices
No n8n / Make.com / Zapier
Python + GitHub Actions + Notion MCP provides the same automation with zero cost, full control, and no middleware to maintain.
FINAL
Embed Signals in Existing Schema
Instead of a separate Signals database, signal data is added as fields in existing Companies and Contacts tables. 90% value at 30% complexity.
FINAL
Lead Score v1 Before v2
Don't change weights until signals data exists. Giving 25% weight to zero-value fields weakens the formula. Run v1 first, evaluate, then iterate.
FINAL
Run Existing Code First
daily_sync.py and lead_score.py are built but never executed. Any new development before running these is a distraction from revenue.
FINAL
4 Phases, Not 7
Phase inflation creates illusion of progress. Multiple "phases" were actually steps within a single day's work. Reduced to 4 actionable phases.
FINAL
80%+ COLD is Expected
Cold Apollo data with no outreach history will score low. This is correct behavior, not a bug. Calibration validates this.
FINAL
No Claude AI Layer Yet
Claude integration for automated analysis is valuable but premature. Need real data flowing first. Deferred to post-Phase 4.
REVISIT
News API Not Worth It
Apollo has no news endpoint. External API (Google News, Crunchbase) adds complexity without proven ROI. Deferred indefinitely.
FINAL
Safe Boolean Writing
Only write engagement checkboxes (Email Sent, Replied, etc.) if Apollo explicitly returns the field. Prevents overwriting manually-set True with False.
FINAL
Unified Constants File
All field names, thresholds, and SLA hours in constants.py -- single source of truth. No hardcoded strings scattered across scripts.
FINAL
Action Ready 5-Condition Gating
Before any task creation: Score >= 50, DNC = False, Outreach Status OK, Stage not Customer/Churned, has contact method. Prevents tasks for bad leads.
FINAL
