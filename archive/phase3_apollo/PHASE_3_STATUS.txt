📊 SYNC SPECIFICATIONS
═════════════════════════════════════════════════════════════════════════════

Sync Strategy:
  • Pull-based (not webhook)
  • APScheduler background task
  • 5-minute intervals
  • Incremental updates (timestamp-based)
  • Auto-deduplication (by email)

Data Flow:
  Apollo API (contacts/companies)
    ↓
  Apollo Client (fetch 100 contacts)
    ↓
  Deduplication Logic (check by email)
    ↓
  Notion Client (create/update pages)
    ↓
  Notion Databases (Contacts + Companies)
    ↓
  Logging System (audit trail)

Performance:
  • Pull operation: 1-2 seconds
  • Sync cycle: 15-30 seconds
  • Memory usage: ~50-100 MB
  • Deduplication: O(1) lookup per contact

Reliability:
  • Automatic retry (3 attempts per contact)
  • Graceful error handling
  • Detailed logging
  • Timestamp tracking for incremental sync
